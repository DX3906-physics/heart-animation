# 🚀 GitHub Pages 部署指南

## 📋 步骤说明

### 第一步：创建GitHub账号
1. 访问 [https://github.com](https://github.com)
2. 点击 "Sign up" 注册新账号
3. 验证邮箱

### 第二步：创建新仓库
1. 登录GitHub
2. 点击右上角的 "+" 号 → "New repository"
3. 仓库名称输入：`heart-animation`
4. 选择 **Public**（公开）
5. 勾选 "Add a README file"
6. 点击 "Create repository"

### 第三步：上传文件
1. 在新创建的仓库页面，点击 "uploading an existing file"
2. 将以下文件拖拽到上传区域：
   - `index.html` ⭐（主要文件）
   - `README.md`
   - `.gitignore`
   - `.github/workflows/pages.yml`
3. 填写提交信息："Add heart animation files"
4. 点击 "Commit changes"

### 第四步：开启GitHub Pages
1. 在仓库页面，点击 "Settings" 标签
2. 左侧菜单找到 "Pages"（页面）
3. Source（源）选择 "Deploy from a branch"
4. Branch（分支）选择 "main" 和 "/ (root)"
5. 点击 "Save"

### 第五步：获取您的链接
等待几分钟，GitHub会生成您的专属链接：
```
https://[您的用户名].github.io/heart-animation/
```

## 🎯 示例链接格式
- 如果您的用户名是 `alice2024`
- 那么链接就是：`https://alice2024.github.io/heart-animation/`

## ⏱️ 部署时间
- 首次部署需要 **5-10分钟**
- 后续更新只需要 **1-2分钟**

## 📱 分享您的作品
获得链接后，您可以：
- 微信分享给朋友
- 发布到社交媒体
- 添加到个人作品集

## 🔧 自定义域名（可选）
如果您有自己的域名，可以绑定到GitHub Pages

## 💡 常见问题

**Q: 文件上传失败怎么办？**
A: 确保所有文件都正确创建，可以重试

**Q: 页面打不开怎么办？**
A: 等待几分钟再试，或检查仓库设置

**Q: 如何更新内容？**
A: 直接修改文件并重新上传即可
